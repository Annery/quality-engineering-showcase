# 1. Задача по написанию тест-кейсов

**Условие**: Веб-приложение позволяет пользователю оплатить заказ банковской картой. Оплата происходит через внешний платежный шлюз после нажатия кнопки «Оплатить»:

1. Выполняется клиентская валидация
2. Данные отправляются в платежный шлюз
3. Возможен редирект на 3-D Secure
4. Платёжный шлюз возвращает результат:
	- Успешно
	- Отклонено банком
	- Ошибка платёжного сервиса
5. Приложение обновляет статус заказа
6. Пользователю отображается результат оплаты
7. На email отправляется чек (при успешной оплате)

Необходимо написать тест-кейсы, покрывающие:
	- позитивный сценарий
	- негативный сценарий
	- граничные значения
	- проверка валидации
	- проверка интеграции с платежным шлюзом

Ожидаемый результат: Тест кейсы в формате:
	- ID
	- Название
	- Предусловия
	- Шаги
	- Ожидаемый результат
  
* * *

## 1.1. Тест‑кейсы для оплаты банковской картой через платёжный шлюз

### 1.1.1. Допущения и контекст

**Сущности и статусы**

- Есть заказ `ORDER-<id>` со статусами (например):  
    `CREATED` -> (`PAID` | `DECLINED`| `PAYMENT_ERROR`).
- В системе есть страница «Оплата заказа» с кнопкой «Оплатить».
- После нажатия «Оплатить» происходит:
    1.  клиентская валидация полей;
    2.  отправка данных во внешний платёжный шлюз;
    3.  редирект на 3‑D Secure (при необходимости);
    4.  возврат результата от шлюза (успех/отклонено/ошибка);
    5.  обновление статуса заказа;
    6.  отображение результата пользователю;
    7.  чек на email (только при успехе).

**Платёжная форма**

- Поля: `Номер карты (PAN)`, `Срок действия (MM/YY)`, `CVV/CVC`, `Имя держателя карты`.
- Email пользователя для чека уже известен из профиля/заказа.
- Поле ввода PAN ограничено вводом до 19 цифр. Допустим ввод только цифр.

| Система | Длина PAN |
|---|---|
| Visa | 13, 16, 19 |
| Mastercard | 16 |
| American Express | 15 |
| UnionPay | 16-19 |

**Тестовые окружения**

- Для интеграции нужен мок/стаб платёжного шлюза, позволяющий принудительно получать ответы:
    - `SUCCESS`
    - `DECLINED_BY_BANK` (например, недостаточно средств/карта заблокирована)
    - `PAYMENT_SERVICE_ERROR` (например, 5xx/таймаут/ошибка подписи).

**Нефункциональные ожидания**

- Во время оплаты показывается индикатор «Обработка…», повторное нажатие «Оплатить» блокируется.
- Система ведёт логирование (без сохранения CVV и PAN в логи).

### 1.1.2. Тест‑кейсы

#### PAY-001. Успешная оплата без 3-D Secure

**Предусловия**

- Существует заказ `ORDER-1` в статусе `CREATED`.
- Доступны валидные тестовые реквизиты карты, для которых шлюз вернёт `SUCCESS` без 3‑D Secure.
- Email пользователя известен в заказе (например, `user@example.com`).

**Шаги**

1.  Открыть страницу оплаты заказа `ORDER-1`.
2.  Ввести валидные данные карты.
3.  Нажать кнопку «Оплатить».
4.  Дождаться ответа платёжного шлюза.

**Ожидаемый результат**

- Клиентская валидация пройдена. Запрос к шлюзу отправлен, шлюз возвращает `SUCCESS`.
- Статус заказа обновлён на `PAID`.
- Пользователю показано подтверждение успешной оплаты (например, на UI показан «Оплата успешна»).
- На email пользователя отправлен чек.

#### PAY-002. Успешная оплата с 3-D Secure

**Предусловия**

- Существует заказ `ORDER-2` в статусе `CREATED`.
- Доступна валидная тестовая карта, требующая 3-D Secure.
- Доступна тестовая страница Secure и тестовый пароль/код известен.
- Email пользователя известен в заказе (например, `user@example.com`).

**Шаги**

1.  Открыть страницу оплаты заказа `ORDER-2`.
2.  Ввести валидные данные карты.
3.  Нажать кнопку «Оплатить».
4.  Убедиться, что произошёл редирект/открылась страница 3-D Secure.
5.  Пройти аутентификацию (ввести пароль/код/подтвердить).
6.  Дождаться возврата на сайт веб-приложения и ответа платёжного шлюза.

**Ожидаемый результат**

- Клиентская валидация пройдена. Запрос к шлюзу отправлен.
- После успешного прохождения 3D-Secure шлюз возвращает `SUCCESS`.
- Статус заказа `ORDER-2` обновлён на `PAID`.
- Пользователю показано подтверждение успешной оплаты именно для заказа `ORDER-2` (например, на UI показан «Оплата успешна»).
- На email пользователя отправлен чек.

**Шаги**

1.  Открыть страницу оплаты заказа `ORDER-12`.

3.  Нажать кнопку «Оплатить».
4.  Дождаться ответа/ошибки платёжного шлюза.

**Шаги**

**Ожидаемый результат**

# 3. Задача по теории вероятности
**Условие**: Монету подбрасывают три раза. Какова вероятность того, что
выпадет ровно два орла?
**Ожидаемый результат**: Описание решения и ответ

### Допущения

1.  Монета **честная**, т.е. в каждом броске:
	- орел (О) = 1/2
	- решка (Р) = 1/2
2. Броски считаются **независимыми**: результат второго не зависит от результата первого и т.д.


* * *

### Решение

Монета бросается 3 раза. У каждого броска 2 исхода - О или Р.
Всего исходов - `2 * 2 * 2 = 2^3 = 8`.
Подходящих комбинаций выпадения 2х орлов при 3х бросках всего 3: ООР, ОРО, РОО. 
Вероятность их выпадения: 3/8 = 0,375 или 37,5%.

Или по формуле Бернулли:
`P = C(n,k) * p^k * (1-p)^(n-k)`

n = 3 (количество бросков)
k = 2 (успех - 2 орла)
p = 1/2 (вероятность орла)
С(n,k) - число комбинаций

`C(n,k) = n! / k! * (n-k)!`

С(n,k) = 3! / 2! * (3-2)! = 6 / 2 * 1 = 3

P = 3 * 1/2^2 * (1-1/2)^(3-2) = 3 * 1/4 * 1/2 = 3/8 = 0,375
